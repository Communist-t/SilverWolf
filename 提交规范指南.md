# 项目提交完整注意事项

> 通用开发 / 毕业设计 / 公司项目都适用

---

## 一、提交前必做检查清单

### 1. 代码完整性检查

1. 项目能**本地一键运行**，无缺失依赖、路径报错
2. 环境配置文件有模板（示例配置），不直接提交真实密钥
3. 数据库脚本齐全：建表语句、基础测试数据、说明文档
4. 第三方依赖说明：`requirements.txt` / `pom.xml` / `package.json` 完整
5. 版本统一：JDK / Python / Node / 数据库版本写清，避免跑不起来
6. 注释齐全：核心逻辑、接口、复杂算法加注释，README 清晰

### 2. 文档配套齐全（提交必备）

- README：项目介绍、部署步骤、启动命令、账号密码、功能模块
- 设计文档：需求文档、ER 图、流程图、接口文档
- 使用手册：操作步骤、截图、功能演示说明
- 测试文档：测试用例、测试结果、BUG 修复记录

### 3. 格式与命名规范

- 文件夹命名清晰，不要中文、空格、特殊符号
- 压缩包命名统一：`姓名-项目名称-版本`
- 代码缩进统一，无大量冗余注释、调试打印代码
- 文件不要多层嵌套，目录结构一目了然

### 4. 运行测试

换一台干净环境重新拉取项目部署，确认：

- 不用额外安装未知软件
- 无硬编码本地绝对路径（`D:/xxx`、`/Users/xxx`）
- 端口冲突、跨域、文件上传路径问题提前处理

## 二、绝对不能提交的文件 / 内容

### （一）隐私、密钥、账号类

- 数据库真实账号密码、Redis 密码、MQ 密钥
- 阿里云 / 腾讯云 / OSS / COS API Key、Secret
- JWT 加密私钥、微信 / 支付宝支付商户密钥
- SSH 私钥、服务器登录密码、数据库连接真实配置

**解决方案**：只提交 `config.example.yml` / `.env.template`，真实配置加入 `.gitignore`

### （二）编译产物、缓存、日志

| 类型 | 排除内容 |
|------|----------|
| Java | `target/`、`.class`、`.jar`、`.war` |
| Python | `__pycache__/`、`.pyc`、`dist/`、`build/` |
| 前端 | `node_modules/`、`dist/`、`.vite/`、`.nuxt/` |
| C# | `bin/`、`obj/`、`Debug/`、`Release/` |
| 日志 | `logs/`、`*.log` |
| 缓存 | `.idea/`、`.vscode/`、`.DS_Store`、`Thumbs.db` |

### （三）本地环境、个人配置

- IDE 专属配置：`.idea/`、`.iml`、`.vscode/settings.json`、`.project`
- 本地系统文件：Mac `.DS_Store`、Windows 缩略图缓存
- 本地数据库文件：`.mdb`、本地 SQLite 库、本地存储的上传图片

### （四）超大无关资源

- 安装包、软件安装程序（JDK、MySQL、Node 安装包）
- 高清视频、大量测试图片、备份压缩包
- 重复备份文件夹（`项目备份`、`旧版本`、`测试副本`）
- 几十 G 数据集、本地下载的第三方素材

### （五）敏感业务、违规内容

- 真实业务数据：客户手机号、身份证、订单隐私数据
- 测试时爬虫抓取的他人隐私、版权图片 / 视频
- 色情、暴力、侵权素材、未授权商用图片字体
- 内部公司涉密文档、未公开业务方案

### （六）调试垃圾代码

- 大量注释掉的废弃代码、临时测试函数
- 打印调试日志 `print("测试123")`、`console.log("debug")`
- 临时写死的本地路径、本地 IP、测试死账号
- 遗留 TODO、未完成的半成品功能代码（无注释说明）

### （七）其他禁止提交内容

- 操作系统生成的隐藏文件、回收站文件
- 证书、SSL 真实证书文件（`.crt` / `.key` 生产证书）
- 付费第三方 SDK 完整安装包（只留依赖配置，不上传 SDK 本体）
- 个人简历、私人文档、聊天截图等无关文件

## 三、通用规避方案

Git 提交和压缩包提交都适用以下方案：

### 1. 使用 `.gitignore`

Git 项目新建 `.gitignore`，配置好上述不需要提交的目录和文件类型。

### 2. 打包前清理

- 前端执行 `npm run clean`
- Java 执行 `mvn clean`
- Python 删除 `__pycache__` 等缓存文件夹
- 删除 `logs/`、本地数据库、编辑器文件夹

### 3. 配置分离

所有账号密钥写示例模板，真实配置不打包。

### 4. 打包前二次浏览目录

确认无超大文件、无隐私信息、无编译产物。

## 四、不同场景额外注意

### 1. 毕业设计提交

- **不能提交**：大型安装包、几万行第三方源码、真实用户隐私数据
- **必须提交**：完整源码、部署文档、设计说明书、演示截图

### 2. 公司 Git 仓库提交

- **严禁提交**：生产密钥、线上数据库地址、客户真实数据、服务器登录信息
- **严禁提交**：未完成功能、有漏洞测试代码、硬编码密码

### 3. 线上平台 / 竞赛项目提交

- **不能上传**：超过限制大小的视频、本地数据集、付费资源
- **不能包含**：侵权素材、爬虫抓取的他人私有数据
