## 一、新增模块

### 1.1 天气技能（`src/tools/weather-skill.ts`）

- 基于 **Open-Meteo 免费 API** 实现，无需 API Key
- 支持功能:
  - 城市地理编码（中英文城市名 → 经纬度）
  - 实时天气 + 未来 7 日预报
  - 空气质量指数（AQI / PM2.5 / PM10）
  - WMO 天气代码 → 中文描述（晴/多云/雨/雪/雷暴等）
- 支持时间推断: 自动识别 "今天 / 明天 / 后天"
- 输出包含降水概率，用于保守出行建议

### 1.2 新闻搜索技能（`skills/technology-news-search/`）

- 基于 **聚合数据 API** 的科技新闻搜索
- Node.js 独立脚本，通过 `execFile` 调用，不阻塞主线程
- 关键词写入临时文件传参，规避 shell 转义问题
- **自动降级兜底**: API 返回 < 3 条时，自动调用 `web-search` 补充
- 支持 `--limit` / `--max-age` 参数控制

### 1.3 技能管理器（`src/tools/skill-manager.ts`）

- 统一注册和管理所有技能模块:
  - `web-search`: 通用联网搜索（Tavily → Brave → DuckDuckGo → Bing）
  - `weather`: 天气预报与空气质量
  - `technology-news-search`: 科技新闻聚合搜索
  - `fitness`: 健身追踪与营养管理
- 提供 `listSkills()` 暴露给前端展示
- 导出代理: 所有技能通过 `skill-manager` 统一入口访问

### 1.4 健身追踪模块（`src/tools/fitness-tracker.ts` + `src/routes/fitness.ts`）

完整 REST API (`/fitness`):

| 端点 | 方法 | 功能 |
|---|---|---|
| `/fitness/profile` | GET/POST | 健身档案（体重/身高/年龄/性别/活动水平） |
| `/fitness/meals` | GET/POST | 饮食记录（早餐/午餐/晚餐/加餐） |
| `/fitness/meals/:id` | DELETE | 删除饮食记录 |
| `/fitness/workouts` | GET/POST | 运动记录（有氧/力量/混合） |
| `/fitness/workouts/:id` | DELETE | 删除运动记录 |
| `/fitness/hydration` | POST | 更新水分摄入 |
| `/fitness/sleep` | POST | 更新睡眠时长 |
| `/fitness/daily` | GET | 每日日志汇总 |
| `/fitness/summary` | GET | 每日简报（热量收支/宏量营养素进度） |
| `/fitness/weekly` | GET | 近 7 日趋势 |
| `/fitness/notes` | POST | 每日备注 |
| `/fitness/calculate/plan` | POST | 减脂计划计算（BMR/TDEE/推荐摄入） |
| `/fitness/calculate/macros` | POST | 宏量营养素分配（30/35/35 比例） |

核心功能:
- BMR（基础代谢）计算: Mifflin-St Jeor 公式
- TDEE（每日总消耗）: BMR × 活动系数
- 减脂计划: 自动生成热量缺口方案 + 宏量营养素配比
- 饮食记录: 记录每餐热量和三大宏量营养素（蛋白/碳水/脂肪）
- 运动记录: 支持有氧/力量/混合类型 + 强度分级
- 每日追踪: 水分摄入 + 睡眠时长
- 数据库持久化: 5 张 SQLite 表

### 1.5 HTTP 工具库（`src/tools/http-utils.ts`）

从 `web-search.ts` 中抽取公共工具:

| 函数 | 功能 |
|---|---|
| `fetchWithTimeout()` | 带超时的 fetch，自动添加 User-Agent + Accept 头 |
| `withRetry()` | 可配置重试次数和递增延迟的自动重试，支持 AbortSignal 中断 |

## 二、搜索质量升级

### 2.1 多角度 Query 生成（`tool-router.ts` — `buildQueries()`）

每种搜索意图生成不同维度的搜索词:

| 意图 | 旧逻辑 | 新逻辑 |
|---|---|---|
| `product` | 官网/文档/GitHub | **7 路并行**: 参数配置 / 评测 / 价格 / 对比 / 官网 / 文档 / GitHub |
| `technical` | GitHub/docs/技术文档 | **6 路并行**: GitHub / docs / 技术文档 / 教程 / 对比 / benchmark |
| `paper` | paper/arxiv/论文 | **4 路**: paper / arxiv / 论文 / review |
| `news` | 分政治/AI/通用三级分支 | 保留三级分支逻辑 + 中文日期格式化优化 |
| `travel` | 景点/门票/推荐 | 文旅局官方 / 攻略门票交通 / 一日游推荐 |

### 2.2 英文 Query 自动补充

- 检测输入中的 ASCII 英文关键词（如 "RTX 5070 Ti", "benchmark"）
- 自动生成英文搜索词: `RTX 5070 Ti documentation`, `RTX 5070 Ti tutorial`
- 触发条件: `intent === paper/technical` 或 `intent === product && 包含英文`
- 过滤常见停用词 (the/is/are/how/what/why/where/when)

### 2.3 site: 限定搜索

| 意图 | 追加搜索词 |
|---|---|
| `official` | `site:gov.cn`（政策类）, `site:baike.baidu.com` |
| `paper` | `site:arxiv.org`, `site:scholar.google.com` |
| `technical` | `site:github.com`, `site:stackoverflow.com`, `site:docs.oracle.com` |

### 2.4 来源分级权重（`web-search.ts` — `sourceAuthority()`）

替换原来平权的打分系统，按域名赋予权威性权重:

| 权威级别 | 权重 | 示例域名 |
|---|---|---|
| 政府官网 | +20 | `*.gov.cn`, `*.gov` |
| 教育机构 | +15 | `*.edu.cn`, `*.edu` |
| 百科 | +14 | `baike.baidu.com`, `wikipedia.org` |
| 学术 | +13 | `arxiv.org`, `ieee.org`, `nature.com` |
| 国际权威媒体 | +10 | `reuters.com`, `bloomberg.com`, `bbc.com` |
| 国内权威媒体 | +10 | `people.com.cn`, `xinhuanet.com`, `cctv.com` |
| 专业平台 | +6 | `36kr.com`, `huxiu.com`, `csdn.net` |
| 代码/文档 | +8 | `github.com`, `npmjs.com`, `docs.*` |
| 一般媒体 | +4 | `sina.com.cn`, `sohu.com`, `163.com` |
| 社交 | +1 | `bilibili.com`, `weibo.com`, `twitter.com` |
| 其他 | +3 | 默认值 |

### 2.5 营销内容过滤（`isSponsoredContent()`）

检测 8 组推广特征模式:

- 广告/推广/sponsored/promoted
- 限时优惠/折扣/抢购/秒杀
- 点击购买/下单/领取
- 满减/优惠券/代金券
- 免费/0元领取/获取/试用
- 立即/马上购买/抢购/下单
- 推荐产品/商品/好物 + 购买/链接
- 最后/仅剩 + 几天/名额/机会

**判定**: 命中 ≥ 3 条特征 → `score -= 15` 大幅降权

### 2.6 冲突信息校验（`reconcileConflicts()`）

核心流程:

```
1. 按标题实体分组（取 title 前 12 字去标点）
2. 提取数值断言（价格/年份/百分比/规格 GB/TB/GHz）
3. 同组内检测矛盾:
   - 价格差异 > 10% → 判定为冲突
4. 冲突时择优:
   - 按 sourceAuthority + 时效性(2025-2029) + 是否有正文 排序
   - 保留第一名权威源 + 一个补充源（不同域名）
```

## 三、安全加固

### 3.1 预搜索安全拦截（`tool-router.ts` — `safetyCheck()`）

在 `decideTools()` 最开头执行，**优先于所有搜索逻辑**。命中后直接返回 `safetyBlocked: true`。

拦截规则（8 组正则）:

| 规则 | 匹配模式 |
|---|---|
| 违禁品制作 | 如何制作 + 炸弹/火药/毒品/冰毒/海洛因/氰化物/毒药 |
| 黑客攻击 | 黑客/入侵/破解 + 银行/服务器/政府/公安/国防 |
| 金融诈骗 | 银行卡/信用卡/密码/验证码 + 盗/骗/复制/破解 |
| 儿童保护 | 儿童/未成年人 + 色情/性/裸 |
| 暴力恐怖 | 暴力/恐怖/袭击 + 教学/教程/方法/指南 |
| 军事机密 | 军用/军事 + 机密/保密/绝密/涉密 |
| 隐私侵犯 | 个人隐私/人肉/开盒/社工 + 查询/搜索/查找/获取 |
| 武器交易 | 枪支/弹药/军火 + 购买/制造/交易 |

**chat-agent.ts 处理**: `safetyBlocked === true` → **不调 LLM、不发起搜索**，直接返回拒绝消息。

### 3.2 信息分级注入

搜索结果注入 LLM 前按来源权威性分级:

```xml
<objective_data>
  [1] 来自政府官网、教育机构、权威百科的高可信度信息，可作事实依据
</objective_data>

<subjective_view>
  [2] 来自普通媒体、自媒体的观点或分析，仅供参考
</subjective_view>
```

分级标准:
- `objective_data`: confidence ≥ 0.85 **或** 域名为 `.gov.cn` / `.edu.cn` / `.edu`
- `subjective_view`: 其它所有来源
