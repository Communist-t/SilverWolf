# 银狼 Agent 命令手册

> 项目根目录：`silver-wolf-agent/`
>
> 三层架构：前端（静态页面）+ 后端（Node.js API）+ 数据库（PostgreSQL）

---

## 架构总览

```
┌──────────────────┐       ┌──────────────────┐       ┌──────────────────┐
│     前端 (8080)   │──────▶│    后端 (3000)    │──────▶│  PostgreSQL(5432) │
│  静态 HTML/CSS/JS │  HTTP │   Hono + Node.js  │  TCP  │   postgres:16     │
│  Nginx / serve    │       │   TypeScript API  │       │   Docker 容器     │
└──────────────────┘       └──────────────────┘       └──────────────────┘
```

| 层 | 技术栈 | 端口 | 代码位置 |
|----|--------|------|----------|
| 前端 | 纯静态 HTML/CSS/JS + Nginx | `8080` | `public/` |
| 后端 | Node.js 20 + Hono + TypeScript | `3000` | `src/` |
| 数据库 | PostgreSQL 16 Alpine | `5432` | `db/init.sql` |

## 一、前置准备

### 1.1 安装依赖

```bash
npm install
```

### 1.2 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`，关键配置：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `LLM_API_KEY` | 大模型 API Key（留空则使用数据库中配置的模型） | 空 |
| `LLM_BASE_URL` | 模型 API 地址 | `https://api.openai.com/v1` |
| `LLM_MODEL` | 模型名称 | `gpt-4o` |
| `HOST` | 后端监听地址 | `127.0.0.1` |
| `PORT` | 后端监听端口 | `3000` |
| `DATABASE_URL` | PostgreSQL 连接字符串 | `postgres://silverwolf:silverwolf_dev_2026@127.0.0.1:5432/silver_wolf_agent` |
| `WEB_SEARCH_PROVIDER` | 联网搜索引擎（`auto`/`tavily`/`brave`/`html`） | `auto` |

## 二、数据库启动

### 2.1 Docker 容器启动（唯一方式）

```bash
docker compose up -d postgres
```

- 使用 `postgres:16-alpine` 镜像
- 容器名：`silver-wolf-postgres`
- 首次启动自动执行 `db/init.sql`（13 张表）
- 数据持久化到 Docker volume `pg_data`
- 健康检查：每 5 秒执行 `pg_isready`

### 2.2 验证数据库

```bash
# 查看容器状态
docker compose ps postgres

# 查看表结构
docker exec silver-wolf-postgres psql -U silverwolf -d silver_wolf_agent -c "\dt"

# 查看模型配置
docker exec silver-wolf-postgres psql -U silverwolf -d silver_wolf_agent \
  -c "SELECT id, label, provider, model, active FROM llm_model_configs ORDER BY active DESC;"
```

### 2.3 数据库连接信息

| 项 | 值 |
|----|----|
| Host | `127.0.0.1`（宿主机）/ `postgres`（Docker 内部网络） |
| Port | `5432` |
| 用户名 | `silverwolf` |
| 密码 | `silverwolf_dev_2026` |
| 数据库名 | `silver_wolf_agent` |

### 2.4 数据库停止 / 清除

```bash
# 停止数据库
docker compose stop postgres

# 停止并删除容器（数据卷保留）
docker compose down

# 彻底清除（⚠️ 删除所有数据）
docker compose down -v
```

## 四、前端启动

> 前提：后端已在 `http://localhost:3000` 运行。

### 4.1 本地开发模式（静态文件服务器）

```bash
npx serve -l 8080 public
```

- 在 `public/` 目录启动静态文件服务
- 监听 `8080` 端口
- 通过 `public/config.js` 中的 `apiBase` 指向后端 `http://localhost:3000`

### 4.2 Docker 容器模式

```bash
docker compose up -d frontend
```

- 使用 `nginx:alpine` 提供静态文件服务
- Nginx 配置：`frontend/nginx.conf`
- 启动脚本 `frontend/docker-entrypoint.sh` 通过环境变量 `API_BASE` 注入后端地址
- 映射 `8080` → 容器 `80`

### 4.3 前端配置说明

`public/config.js` 控制前端请求后端的地址：

```js
window.SILVER_WOLF_CONFIG = {
  apiBase: window.__SILVER_WOLF_API_BASE__ ?? "http://localhost:3000",
  frontendBase: window.__SILVER_WOLF_FRONTEND_BASE__ ?? "",
};
```

| 场景 | `apiBase` 值 |
|------|-------------|
| 本地开发（前端 8080，后端 3000） | `http://localhost:3000` |
| Docker 全量部署（Nginx 代理） | 容器启动时通过 `API_BASE` 环境变量注入 |
| 同源部署（前端后端同一端口） | 空（同源访问） |

### 4.4 前端页面路由

| URL | 页面 |
|-----|------|
| `http://localhost:8080/` | 落地页（`landing.html`） |
| `http://localhost:8080/login` | 登录页（`login.html`） |
| `http://localhost:8080/chat` | 聊天页（`index.html`） |

## 五、全量 Docker 部署（三容器一键启动）

```bash
docker compose up -d --build
```

三个容器按依赖顺序启动：

```
postgres（健康检查通过） → backend → frontend
```

| 容器 | 服务 | 端口映射 | 依赖 |
|------|------|----------|------|
| `silver-wolf-postgres` | PostgreSQL 16 | `5432:5432` | — |
| `silver-wolf-backend` | Node.js API | `3000:3000` | postgres (healthy) |
| `silver-wolf-frontend` | Nginx | `8080:80` | backend |

容器间通信注意：
- 后端容器内通过服务名 `postgres` 连接数据库（`docker-compose.yml` 已覆盖 `DATABASE_URL`）
- 后端容器内监听 `0.0.0.0`（`docker-compose.yml` 已覆盖 `HOST`）
- 前端容器通过 `API_BASE` 环境变量指向后端

### 仅启动/停止部分服务

```bash
# 只启动数据库
docker compose up -d postgres

# 停止前后端，保留数据库
docker compose stop backend frontend

# 重启后端
docker compose restart backend
```

## 六、日志查看

> 使用 Docker 部署后，所有服务日志通过 `docker compose logs` 或 `docker logs` 查看。

### 6.1 查看所有服务的实时日志

```bash
# 实时滚动查看所有容器的日志
docker compose logs -f
```

### 6.2 只看某个服务的日志

```bash
# 只看后端日志（聊天请求、LLM 调用、错误信息等）
docker compose logs -f backend

# 只看前端日志（页面请求等）
docker compose logs -f frontend

# 只看数据库日志
docker compose logs -f postgres
```

### 6.3 查看最近的日志

```bash
# 只看最后 100 行
docker compose logs --tail 100 backend

# 看最近 5 分钟的日志
docker compose logs --since 5m backend

# 看最近 1 小时的日志
docker compose logs --since 1h backend
```

### 6.4 用容器名直接查看

```bash
docker logs -f silver-wolf-backend
docker logs -f silver-wolf-frontend
docker logs -f silver-wolf-postgres
```

### 6.5 日志查看技巧

| 场景 | 命令 |
|------|------|
| 日常开发最常用 | `docker compose logs -f backend` |
| 排查聊天问题 | 发消息后盯着 `backend` 日志 |
| 排查页面加载问题 | `docker compose logs -f frontend` |
| 排查数据库连接 | `docker compose logs -f postgres` |
| 只看错误日志 | `docker compose logs -f backend 2>&1 \| grep -i error` |

> 按 `Ctrl+C` 退出日志查看（不会停止服务）。

## 七、完整本地开发启动流程

三个终端分别执行：

```bash
# 终端 1：启动数据库
docker compose up -d postgres

# 终端 2：启动后端
npm run dev

# 终端 3：启动前端
npx serve -l 8080 public
```

访问 `http://localhost:8080/` 即可使用。

> ⚠️ **注意**：如果已通过 `docker compose up -d` 启动了全部容器，**不需要**再手动启动后端和前端，否则会导致端口冲突。

## 九、常用运维命令速查

```bash
# === Docker 容器管理 ===
docker compose ps                          # 查看所有容器状态
docker compose stop                        # 停止全部容器
docker compose down                        # 停止并删除容器（保留数据）
docker compose down -v                     # 停止并删除容器和数据卷（⚠️）

# === 数据库操作 ===
docker exec -it silver-wolf-postgres psql -U silverwolf -d silver_wolf_agent  # 进入 psql 交互

# === 健康检查 ===
curl http://localhost:3000/health          # 后端健康检查
docker inspect --format='{{.State.Health.Status}}' silver-wolf-postgres  # 数据库健康状态
```
