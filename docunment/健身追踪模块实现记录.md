# 健身追踪模块 — 实现记录

> 日期：2026-07-04
> 配比方案：100=30+35+35（蛋白质30% / 碳水35% / 脂肪35%）偏减脂方向

---

## 概述

为银狼 Agent 新增健身与营养追踪功能。采用 **Mifflin-St Jeor BMR 公式**计算基础代谢，按 **BMR 踩线吃**的热量策略，配合 **30/35/35 宏量营养素配比**，实现饮食、运动、睡眠和水分的全量追踪。

支持通过自然语言对话记录数据（如"吃了早餐350卡"、"跑步30分钟"），也提供完整的 REST API。

## 数据库设计

### 表结构

```
fitness_profile (1 条/用户)
├── owner_id TEXT PK          — 用户标识
├── bmr INTEGER               — 基础代谢率 (kcal)
├── calorie_target INTEGER    — 每日目标热量 (踩 BMR 线)
├── protein_target_g REAL     — 蛋白质目标 (g)
├── carbs_target_g REAL       — 碳水目标 (g)
├── fat_target_g REAL         — 脂肪目标 (g)
├── weight_kg / height_cm / age / gender / activity_level
└── created_at / updated_at

fitness_daily (1 条/用户/天, UNIQUE owner_id+date)
├── id INTEGER PK
├── owner_id / date
├── calories / protein_g / carbs_g / fat_g    — 由 meals 汇总，自动同步
├── water_ml          — 手动记录
├── sleep_hours       — 手动记录
└── notes

fitness_meals (N 条/用户/天)
├── id INTEGER PK
├── owner_id / date
├── meal_type — 'breakfast' | 'lunch' | 'dinner' | 'snack'
├── food_name / calories / protein_g / carbs_g / fat_g
└── created_at

fitness_workouts (N 条/用户/天)
├── id INTEGER PK
├── owner_id / date
├── type — 'cardio' | 'strength' | 'mixed'
├── duration_minutes
├── details / intensity — 'low' | 'moderate' | 'high'
└── created_at
```

### 自动同步机制

每次 `addMeal()` 调用后自动触发 `syncDailyLog()`，从 `fitness_meals` 聚合当日热量和宏量营养素汇总到 `fitness_daily`。水分和睡眠通过独立接口写入，不做自动清零。

## BMR 计算

使用 Mifflin-St Jeor 公式：

```
男性：BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 + 5
女性：BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 - 161
```

减脂策略：目标热量 = BMR（不作额外活动系数加成），配合 30/35/35 配比。

### 验证数据

输入：70kg / 175cm / 25岁 / 男性
- BMR = 10×70 + 6.25×175 - 5×25 + 5 = 1674 kcal ✓
- 维持热量（久坐 ×1.2）= 2009 kcal
- 目标热量（BMR 踩线）= 1674 kcal
- 蛋白质：1674 × 30% ÷ 4 = 126g
- 碳水：1674 × 35% ÷ 4 = 147g
- 脂肪：1674 × 35% ÷ 9 = 65g
