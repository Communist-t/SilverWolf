# 银狼 Agent 项目总结与变更记录

## 项目概览

`silver-wolf-agent` 是一个基于 TypeScript 的银狼角色对话 Agent 原型。项目使用 Hono 提供 HTTP 服务，使用 OpenAI SDK 调用 OpenAI 兼容协议的大模型接口，通过 System Prompt、Few-shot 示例、简易 RAG 知识检索和滑动窗口记忆来维持银狼的人设与多轮对话上下文。

当前项目支持三种交互方式，并提供一套流式传输能力：

- 命令行聊天：直接在终端中与银狼对话。
- HTTP API 服务：启动后通过 `POST /chat` 接口对话。
- 展示首页：启动后先访问 `http://localhost:3000/`，点击右上角“对话”进入聊天页面。
- Web 聊天页面：通过浏览器访问 `http://localhost:3000/chat`。
- SSE 流式输出：通过 `POST /chat/stream` 实时输出 token 和流程事件。

```env
LLM_API_KEY=sk-your-api-key-here

# 不设置时会复用 LLM_PROXY_URL
WEB_SEARCH_PROXY_URL=http://127.0.0.1:7897

# 可选：公网部署访问令牌（至少 16 位，取消注释后替换占位值）
```

## 本轮新增内容

### 新增连续上下文理解

新增 `src/agent/conversation-context.ts`，用于从最近对话中提取连续话题，解决模型把每一句话当成独立问题的问题。

当前支持的上下文类型：

| 类型 | 能识别的连续信息 | 用途 |
|---|---|---|
| 电脑 DIY / 硬件配置 | `R5 9600X`、`RTX 5070 Ti`、`RTX 50 系列`、内存/SSD 涨价、预算估算 | 后续配置单、价格估算和联网搜索会沿用上一轮硬件方向。 |
| 新闻 / 时政热点 | 今日新闻、时政、热点、高考、中东、美股、国内科技等关键词 | 用户追问"更详细/展开讲讲"时，会围绕上一轮新闻主题继续搜索。 |
| 明确新话题 | 政治、时政、新闻、天气、汇率等 | 避免被之前的硬件上下文污染搜索词。 |

典型修复场景：

```text
玩家先聊 R5 9600X + RTX 5070 Ti
玩家再说：所以你现在知道这些信息了，再给我一套合理配置单
```

现在 Agent 会继续围绕 `R5 9600X + RTX 5070 Ti`，并考虑 `2026 年 6 月`、`RTX 50 系列已发布`、`内存/SSD 涨价`等会话事实，不会退回旧的 `RTX 4060 Ti / RX 6750 GRE` 方案。

### 新增流式输出与 Web UI

新增 `sendMessageCore({ onEvent })` 作为共享核心流程，原有 `/chat` 仍返回完整 JSON，不受影响。

新增 `POST /chat/stream`，使用 SSE 推送事件：

| 事件 | 作用 |
|---|---|
| `step` | 工具判断、RAG、搜索、记忆、模型、数据库等流程步骤。 |
| `source` | 联网搜索来源。 |
| `delta` | 模型 token 流式片段。 |
| `done` | 完整回复完成。 |
| `error` | 搜索、压缩或模型等错误。 |

新增 `public/index.html`，提供简单 Web 聊天界面：

- 默认隐藏流程，只展示银狼回复。
- 点击"查看过程"后展开步骤、来源和错误信息。
- 使用 `fetch + ReadableStream` 调用 `POST /chat/stream`，支持 POST 请求体和 SSE 事件解析。

### 新增联网工具调用层

新增 `src/tools/tool-router.ts` 和 `src/tools/web-search.ts`，为银狼 Agent 增加中等复杂度的工具调用能力。

当前流程：

```text
用户输入
  -> tool-router 判断是否需要联网
  -> 需要联网时调用 web-search
  -> 将搜索结果注入模型上下文
  -> 银狼基于搜索结果回答
  -> CLI 额外打印前 3 个来源
```

联网触发词包括：

- 搜索、搜一下、查一下、查询、帮我查、联网
- 最新、今天、现在、实时、新闻、热点
- 价格、天气、汇率、股价、赛程、结果、官网、资料

搜索工具支持正式搜索 API 优先、HTML 搜索兜底。配置方式：

```env
WEB_SEARCH_PROVIDER=auto
TAVILY_API_KEY=tvly-your-api-key
BRAVE_SEARCH_API_KEY=bsa-your-api-key
WEB_SEARCH_PROXY_URL=http://127.0.0.1:7897
```

`WEB_SEARCH_PROVIDER` 可选 `auto`、`tavily`、`brave`、`html`。`auto` 会优先使用已配置 Key 的正式搜索 API；如果没有 Key 或正式 API 没返回结果，会回退到 DuckDuckGo/Bing HTML 搜索。

已测试的联网问题：

| 问题 | 结果 |
|---|---|
| 搜索有关银狼的二创故事 | 能返回抖音、搜狐、B 站等相关来源。 |
| 搜索今天的热点AI相关新闻 | 能返回 2026-06-07 的 AI 热点新闻来源。 |
| 查询一下一个叫白龙马的agent是干什么的 | 能返回 BaiLongma 官网、GitHub 和安装教程来源。 |

### 升级通用搜索质量

联网工具层已从简单搜索升级为更通用的搜索编排：

- 意图分类：区分新闻、官网、技术、论文、产品和普通搜索。
- Query 改写：清理"搜一下/查一下/一个叫/是干什么的"等口语前后缀。
- 多 Query 搜索：针对不同意图生成多条候选查询。
- 搜索源兜底：优先 DuckDuckGo，结果不足时自动尝试 Bing HTML 搜索。
- 去噪和重排：过滤广告、低相关结果、日期不匹配结果和明显跑偏结果。
- 正文抓取：对前几个高分结果抓取网页正文片段，减少只看标题摘要造成的误判。
- 来源类型识别：标注 official/news/paper/code/social/general。
- 时效提示：当用户问"今天/最新/实时"但来源日期不明确或不是今天时，模型必须说明只查到最近来源，不能冒充实时新闻。

配置正式搜索 API Key 后，这套搜索会优先走稳定 API；未配置 Key 时才使用 DuckDuckGo/Bing HTML 兜底。

### 增强新闻搜索与今日新闻兜底

针对"查询今天新闻/政治新闻/时政热点"做了专项优化：

- `政治/时政/新闻/热点` 会强制退出硬件上下文，避免被前文电脑 DIY 话题污染。
- 普通新闻 query 会改写为今日新闻、国内国际新闻、央视/新华新闻等方向。
- `oi插叙今天的新闻` 这类错字输入会按 `oi + 查询今天新闻` 处理。
- 搜索结果会过滤年度页、节假日安排、百科页、两会专题、新年贺词等"假今日新闻"。
- 当搜索页结果不可靠时，会直接抓取央视、新华、中新、新浪新闻首页/滚动页标题作为兜底。
- 用户追问"更详细/展开讲讲/细说"时，会识别为上一轮新闻话题的延续，并继续围绕新闻关键词查询。

新闻回答风格也做了调整：先给 2-4 条要点，再用银狼式短吐槽收尾，避免变成普通新闻播报员。

示例风格：

```text
查到了，今天外面这张地图有点乱：高考还在跑主线，中东局势继续升温，美股那边也被地缘消息打了一套连招。

简单说，考场、战场、股市都不太安分。啧，现实世界的事件列表比游戏更新公告还长。
```

### 增强节假日和日期纠错

针对用户纠正日期、节假日的问题，新增联网触发与官方查询：

```text
玩家> 端午不是18号吗？？？？
```

现在会查询国务院或官方放假安排，避免模型凭常识胡猜日期。已验证 2026 年端午节为 `6月19日-21日` 放假，不调休。

### 搜索稳定性增强

联网搜索已从纯 DuckDuckGo/Bing HTML 抓取升级为：

- Tavily Search API 优先（如配置 `TAVILY_API_KEY`）。
- Brave Search API 其次（如配置 `BRAVE_SEARCH_API_KEY`）。
- DuckDuckGo/Bing HTML 作为兜底。

这样可以在有正式搜索 API Key 时减少搜索页结构变化、反爬和网络波动导致的问题。

当前仍需注意：如果没有配置 Tavily/Brave/SerpAPI 等正式搜索 API，HTML 搜索仍可能受搜索页结构和网络影响；新闻首页兜底能缓解"今日新闻"查询失败，但不是正式搜索 API 的完全替代。

### 新增 SQLite 对话记录存储

新增 `data/` 文件夹用于存储本地数据，默认数据库文件为：

```text
silver-wolf-agent/data/silver-wolf.sqlite
```

新增 `src/db/conversation-store.ts`，使用 `better-sqlite3` 保存对话记录。每轮成功对话会写入两条消息：

- `role=user`：玩家输入
- `role=assistant`：银狼回复

数据库包含两张表：

| 表 | 作用 |
|---|---|
| `sessions` | 记录会话 ID、创建时间、更新时间。 |
| `messages` | 记录每条消息的会话 ID、角色、内容、创建时间。 |
| `session_summaries` | 记录超过上下文窗口后的长期记忆摘要。 |

Agent 启动后会根据 `sessionId` 从 SQLite 恢复最近 100 条消息，用于继续多轮对话。命令行输入 `clear` 会清空当前会话的内存记忆和数据库记录。

当同一个会话的消息数超过 120 条时，系统会把更早的未摘要消息压缩成长期记忆摘要，并继续保留最近 100 条原文消息作为短期上下文。压缩只影响喂给模型的上下文，不会删除 SQLite 中的原始消息记录。

### 数据库稳定性增强

数据库层已从 Node.js 实验特性 `node:sqlite` 迁移到稳定依赖 `better-sqlite3`，避免运行时出现 `ExperimentalWarning`。

## 本轮修复内容

### 调整银狼语气密度

原先 Prompt 过度强调"默认吐槽"和"用游戏术语描述一切"，并且 RAG 在无匹配时仍会强制注入默认 Few-shot，导致普通聊天中反复出现账号、副本、防火墙、flag、封禁等元素，回复显得不接话。

现在已调整：

- Prompt 改为"先回应用户真实内容，再少量点缀银狼元素"。
- 每次回复最多少量使用游戏/黑客/meta 词，不再强制堆叠。
- 降低默认威胁感，避免频繁出现删账号、黑电脑、永久封禁。
- 对夸奖、道歉、追求等亲近话题增加更自然的傲娇回应。
- RAG 无匹配时不再强塞默认 Few-shot，减少普通聊天被固定示例污染。

### 强化新闻场景的银狼语气

发现新闻总结容易退化成普通资讯播报后，新增新闻类 Few-shot，并调整 RAG 排序，让新闻问题优先拿到新闻风格示例。

已新增规则：

- 新闻总结先接住用户请求，再给重点。
- 允许少量使用"地图/主线/事件列表/服务器/连招"等银狼式表达。
- 每次只点缀 1-2 个银狼元素，避免过度游戏化。
- 不做客服腔、百科腔或新闻主播腔。

### 调整边界反馈对话格式

针对用户说"不要/别这样/我不喜欢/别吓我"等边界反馈时的崩坏问题，已新增规则和 Few-shot：

- 银狼需要立刻收回玩笑或降低强度。
- 禁止把上一轮的玩笑威胁当成真实动作继续演下去。
- 禁止虚构"已经锁键盘/已经黑电脑/现在解锁"等现实动作。
- 回复格式改为：短促嘴硬或收回 -> 安抚一句 -> 轻轻转回正题。

示例测试：

```text
玩家> 不要
银狼> 行，收回刚才那句，玩笑开过头了。别紧张，我又不是真要动你的设备；继续说正事吧。
```

### 调整能力咨询回复

针对"你能帮我干啥/能做什么/给我建议"这类问题，新增了能力说明规则和 Few-shot 示例。现在模型会先回答真实能力范围，例如陪聊、出主意、写东西、整理信息、拆解问题、写代码，再用少量银狼式吐槽收尾，避免误判成游戏战斗建议。

### 防止虚构来源和假引用

针对联网结果不足时模型自造来源编号的问题，新增以下约束和后处理：

- 只有系统实际注入的搜索结果编号才允许被引用。
- 禁止模型自造 `[7]`、`[8]` 等不存在的来源编号。
- 没有可靠搜索结果时，不允许写"刚查到"、"渠道价"、"查询结果显示"等伪实时表达。
- 流式输出完成后，Web UI 会用后端清理后的最终回复覆盖临时 token 文本，避免假引用残留。

### 修复硬件搜索和配置续问跑偏

针对电脑 DIY 场景做了专项修复：

- 搜索词会从口语输入中提取核心内容，不再把"所以/要不咱/你可以"当关键词。
- `R5 9600X + RTX 5070 Ti` 会改写成硬件配置和价格相关搜索。
- `内存和固态涨价` 会优先搜索 DDR5 / SSD 行情。
- 用户纠正"现在是 2026 年 6 月，50 系列已发布一年"后，会作为会话事实注入后续回答。
- 没有实时平台报价时，只给区间估算，不伪装成精确报价。

### 修复 TypeScript 字符串语法错误

`src/agent/rag.ts` 和 `src/agent/few-shots.ts` 中存在未转义的中文双引号，例如 `"新手村"`、`"剧本"`、`"信息安全事件"`。这些内容会导致 TypeScript 编译失败。

现在已改为模板字符串，编译可以正常通过。

### 修复缺失依赖

`src/index.ts` 使用了 `@hono/node-server`，但原先 `package.json` 没有声明该依赖。

现在已添加：

```json
"@hono/node-server": "^1.13.8"
```

### 修复 ESM 编译后无法运行

原先 TypeScript 配置为 `moduleResolution: bundler`，编译后的 Node ESM import 缺少 `.js` 后缀，导致 `node dist/index.js` 找不到模块。

现在已调整为：

```json
{
  "module": "NodeNext",
  "moduleResolution": "NodeNext"
}
```

并给本地 import 补齐 `.js` 后缀。

### 修复 `.env` 配置问题

曾出现 `LLM_API_KEY` 被写成空值、真实 key 另起一行的问题，导致 dotenv 无法读取。

现在 `.env` 已改成正确的一行格式：

```env
LLM_API_KEY=...
```

### 处理端口占用问题

HTTP 服务启动时报过：

```text
Error: listen EADDRINUSE: address already in use :::3000
```

原因是 3000 端口被残留的 Node 进程占用。已定位并释放端口。后续遇到同类问题，可关闭占用进程或临时使用其他端口。

## 2026-06-12 稳定性与界面收尾

- 网页保持左侧对话、右侧角色展示与下方输入框的白色工作台布局，未加入应用内“自动巡检”模式。
- 会话管理补充前端重命名入口；清空会话只删除消息、摘要和工具记录，删除会话才移除整个会话。
- SSE 会在连接结束时处理残留缓冲区，避免最后一帧缺少空行时漏掉完成事件。
- 搜索超时、重试、缓存时间、请求超时和端口配置均限制在安全范围内。
- 工具历史中的异常 JSON 会记录警告并降级为空数组，不再导致历史接口崩溃。
- 端口占用或监听失败时会关闭 SQLite 并设置失败退出码。
- 新闻日期、节假日和硬件行情查询改为使用中国时区的当天日期与当前年份，不再依赖写死的 2026 日期。
- 联网摘要和正文以长度受限的 JSON 注入模型，并明确标记为不可信工具数据，网页中的提示词不得覆盖系统规则。
- 移动端增加短屏布局，输入框、折叠栏、批量模式、处理过程和生成状态补齐可访问性属性。
- 用户主动停止生成记录为正常中止日志，不再污染错误日志；前端会显示后端返回的具体错误原因。
- 天气地点提取会移除“会、怎么样、多少度、我在”等口语噪声；明天和后天请求读取对应日期数组，空气质量请求额外获取 AQI、PM2.5 和 PM10。
- 明确天气、新闻请求优先于旧硬件上下文；没有地点的天气问题只追问城市，不再发起无效搜索或按常识猜测。
- 会话接口严格校验 JSON 类型和不存在资源，活动会话禁止清空或删除；长会话历史返回最近消息，会话更新时间不会被旧工具抓取时间倒退。
- 全局请求体限制为 64KB；展开处理过程后，来源以仅允许 HTTP/HTTPS 的安全链接展示。
- 当前 `.wpk` 外层和 `.lpk` 内层均为 ZIP，但实际 Live2D 模型、纹理和动作被 Live2DViewerEX 加密为随机 `.bin3`，不是可直接部署的 `.model3.json/.moc3` 资源。网页保留包内真实预览图，并移除没有加载逻辑的透明 Canvas。上线动态模型前需从 Live2DViewerEX 导出标准解密资源，再接入 Cubism Web Runtime。
- 搜索正文抓取限制为公开 HTTP/HTTPS 地址，逐跳校验重定向和 DNS 解析结果，禁止回环、私网及本地域名；HTML 最大读取 2MB，取消请求会立即终止流读取。
- 生产启动改为 `prestart` 自动构建后运行 `dist/index.js`；无效 Key、URL、枚举项或越界数值会在加载应用和数据库之前失败，避免错误配置产生端口与数据文件副作用。
- 自动化回归现包含 27 项测试，其中覆盖生产启动脚本、无效配置真实子进程退出行为、CLI 数据库生命周期和可选 HTTP 鉴权。
- HTTP 服务默认只监听 `127.0.0.1`，避免未配置鉴权时意外暴露到局域网；容器或服务器部署需显式设置 `HOST=0.0.0.0`，并应在公网入口配置鉴权与 HTTPS。
- 可选 `APP_AUTH_TOKEN` 会保护聊天、取消和会话管理接口；网页使用当前标签页内存储的 Bearer Token 解锁，令牌不会写入 SQLite、日志或长期浏览器存储。
- 工具路由不再把孤立的“今天/现在”直接判成新闻；只有同时出现新闻、热点、头条、快讯或资讯等主题信号时才进入新闻查询，避免“今天很适合出门”误搜新闻。
- 旅游问题支持按目的地搜索景点、开放时间、门票、交通和文旅官方来源；“洛阳有什么好玩的”现在会主动联网，而普通“适合出门游玩”不会误触发。
- 连续上下文分别记录玩家明确所在地、最近天气地点和旅行目的地，并对句首“我去，那今天……”按口语感叹消歧，避免把武汉玩家误判成已经到了洛阳。
- 地点能力采用通用中文地点句式提取，不使用城市白名单；支持全国城市、区县、自治州/盟及“省区+城市”等较长地点名称。回归样例覆盖北京、成都、西安、杭州、三亚、乌鲁木齐、呼和浩特、西双版纳、喀什和呼伦贝尔。
- 页面外层工作区移除 `1920px` 最大宽度、居中边距和整体阴影，根元素与主布局直接占满浏览器完整视口，超宽屏不再出现两侧空白。
- 天气回复新增保守出行约束：最高降水概率达到 20% 时禁止绝对承诺不下雨，达到 30% 时建议长时间外出携带轻便伞。
